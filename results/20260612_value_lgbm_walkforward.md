# 価値最大化LightGBM v1 walk-forwardバックテスト（実装検証＋第1回実験）

- 日付: 2026-06-12
- 担当: Claude（実装・実験・分析）
- 関連: `results/20260612_value_loss_design.md`（数理設計の正本）、`results/20260612_market_distortion_audit.md`

## 目的

設計書§3-§4の価値最大化カスタム損失（Soft-Kelly対数富＋CE較正アンカー）をLightGBMに実装し、walk-forwardでエッジの有無を検証する。

## 仮説

市場残差パラメータ化 p = softmax(log q + F) と価値最大化損失により、favorite-longshot bias等の市場歪みを拾う賭け選択でROI > 1の領域が見つかる。

## 変更内容（新規実装）

- `src/analysis/market_distortion.py` — 歪み存在証明（§8-5）
- `src/models/value_objective.py` — カスタム目的関数。勾配・GN対角ヘシアンの解析解（§4）。**有限差分との一致を3レジームで検証済み（相対誤差~3e-7）**
- `src/models/train_value_lgbm.py` — walk-forwardハーネス（τ*検証期間チューニング、開催日クラスタブートストラップCI、ベースライン、`--smoke`/`--pure-ce`/`--years`フラグ）

## 使用データ・主要設定

- データ: race_results + race_meta、521,784行（勝者1頭でないレース56件除外）、特徴量37（基本＋shift(1)済み過去走、リーク検査通過）
- 学習: 500ラウンド（CEウォームアップ150→λ: 1→0.2、β: 2→8アニーリング）、κ=0.1、lr=0.05、num_leaves=63
- 検証: 訓練末尾15%日付でτ*選択（Kelly対数富最大化、グリッド0.02〜0.50）
- テスト: 2021〜2026年の6フォールド（2024年は元データ部分欠損、2026年は途中年）

## 原因切り分け診断（純CE、2025年フォールド）

`--pure-ce`（価値項完全無効、500ラウンド全てCE）での再学習:

- CEモデル **2.1086** vs CE市場 **1.9202**
- 決定的事実: 残差パラメータ化では **F≡0（学習ゼロ）でCEモデル=CE市場(1.9202)** になるため、学習後の悪化(+0.19 nats)は**ブースティングが汎化しない残差を過学習した**ことの直接証拠
- 価値項アニーリングあり(2.1346)との差は+0.026にすぎない → **失敗の主因は価値項ではなく、(a)無早期停止の過学習、(b)現特徴量の情報不足**

## 成功/失敗

- インフラ（目的関数・ハーネス・検証手続き）: **成功**（数値検証・健全性検査すべて合格）
- エッジ仮説: **失敗**（プールROI 0.798、CI上限<1.0）。現特徴量セットでは市場較正を超える情報利得がない

## 得られた知見

1. **情報利得ゲート IG := CE市場 − CEモデル > 0 を賭けの前提条件とすべき**。IG ≤ 0 のモデルにいかなるステーキング層を載せても数理的に勝てない（v1はIG = −0.17〜−0.34）
2. 残差学習は初期値（=市場）が既に強いため、無条件のブースティングは悪化しかしない。検証CEによる早期停止が必須
3. 検証手続き（τ*チューニング、CI判定）は「エッジなし」を正しく報告した。手続きの誠実性はこのプロジェクトの資産
