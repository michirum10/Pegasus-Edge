# Data Requirements

Pegasus-Edge で期待値エッジを検証するために、現CSVに不足しているデータと取得優先度を整理する。

## 現在利用可能なデータ

- `race_results.csv`
  - 馬ごとの結果、枠番、馬番、馬名、性齢、斤量、騎手、単勝、人気、馬体重、調教師、馬主など
- `race_meta.csv`
  - `race_id`, `kaisai_date`, `course_type`, `distance_m`

## 不足データ

### P0: バックテストの正確性に必須

- 公式払戻金
- 複勝オッズ
- 複勝払戻金
- オッズ取得時刻

理由:
単勝以外の券種評価ができず、現在は `単勝` と `着順` による近似払戻に依存している。締切前オッズか最終オッズかも明示されていない。

### P1: コンテキストエッジに重要

- 競馬場
- 馬場状態
- 天候
- コース回り、内外、開催日順

理由:
距離短縮、馬場悪化、開催替わり、競馬場バイアスなどの「大衆が過小評価しやすい文脈」を表現するには不足している。

### P2: 時系列特徴量の精度向上

- `horse_id`
- `jockey_id`
- `trainer_id`
- `owner_id`

理由:
現状は `馬名`, `騎手`, `調教師` の文字列で集計している。表記揺れや改名、同名リスクを避けるにはIDが必要。

## 追加時のルール

- 新しいraw列を追加した場合は、まず監査レポートを `results/` に保存する。
- post-race情報は、特徴量ではなく `y` またはバックテスト専用列として隔離する。
- オッズ列は、取得時刻と「締切前/最終」の区別を必ず記録する。
- 既存CSVを上書きせず、取得元と作成日時が分かるファイル名で保存する。
